# Plan Workflow

Create Product Requirements Document and task breakdown.

## PRD Template

Save to `dev/workspace/context/PRD-[feature].md`:

```markdown
# PRD: [Feature Name]

## Problem Statement
What problem are we solving?

## Proposed Solution
How will we solve it?

## Technical Approach
- Technology stack
- Architecture decisions
- Data models

## Risk Assessment
- What could go wrong?
- Dependencies
- Unknowns

## Success Criteria
How do we know it works?
```

## Task List Template

Save to `dev/workspace/context/TASKS-[feature].md`:

```markdown
# Tasks: [Feature Name]

## Phase 1: Setup
- [ ] Task 1
- [ ] Task 2

## Phase 2: Implementation
- [ ] Task 3
- [ ] Task 4

## Phase 3: Testing
- [ ] Task 5
- [ ] Task 6

## Verification Gates
- [ ] All tests pass
- [ ] Code reviewed
- [ ] Documentation updated
```
